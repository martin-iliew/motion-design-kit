# Audit Rules

Use this rubric for report-first and upgrade-first passes.

## Part A: Surface decision table

- weak hero -> `grid-led-hero`
- weak proof near top -> `sticky-proof-strip`
- weak product explanation -> `product-ui-showcase`
- weak features section -> `modular-feature-grid`
- weak integrations proof -> `integration-logo-ecosystem`
- weak pricing clarity -> `comparison-pricing-matrix`
- weak testimonials -> `outcome-testimonial-wall`
- weak objections handling -> `objection-handling-faq`
- weak closing CTA -> `risk-reversal-cta-band`
- weak trust / compliance -> `security-compliance-block`
- weak case-study proof -> `customer-story-metrics`

## Part B: Severity

- `CRITICAL`: confusing offer, absent CTA path, broken hierarchy, unreadable pricing, missing trust on enterprise-like pages
- `WARNING`: weak proof density, generic sections, poor feature grouping, underpowered CTA copy, inconsistent visual system
- `INFO`: additional polish, stronger metric framing, more specific proof labels

## Part C: Audit dimensions

- `MESSAGE`: does the page explain what the product is and why it matters?
- `HIERARCHY`: can the page be scanned in seconds?
- `PROOF`: are logos, metrics, testimonials, trust, or customer evidence present where needed?
- `CONVERSION`: is the CTA path obvious and repeated?
- `POLISH`: does the page look current, intentional, and enterprise-safe without trend-chasing?

## Part D: Hard no list

- brutalism by default
- anti-grid composition
- vague headlines with no audience or outcome
- screenshots without explanation
- testimonial sections with no customer identity or outcome
- pricing cards that hide key differences

## Part E: Quick reference

- default headline structure: product + audience + outcome
- default hero proof: logos, metrics, or a short trust line
- default closing move: CTA band + reassurance
- preferred section rhythm: hero -> proof -> product -> features -> proof -> pricing/faq -> CTA
