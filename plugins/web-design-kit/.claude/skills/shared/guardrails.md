# Guardrails

These rules are mandatory in every mode.

## Conversion

- Above the fold must communicate product, audience, and outcome.
- Primary CTA must be prominent, specific, and repeated later on the page.
- Proof must appear within the first two major scrolls.
- Pricing, objections, or trust gaps must not be replaced by decorative filler.

## Layout

- Use real grid discipline.
- Preserve generous spacing and clear alignment lines.
- Keep section widths and text measures readable.
- Use asymmetry only when it still feels orderly.

## Visual language

- Avoid brutalism, anti-design, maximal collage, and post-friendly novelty by default.
- Avoid gratuitous gradients, noisy backgrounds, chrome-heavy cards, and needless glass.
- Let one visual device lead a section: screenshot frame, proof rail, metric block, or card grid.

## UX and accessibility

- Preserve semantic heading order.
- Keep contrast and body size production-safe.
- Make pricing, testimonials, and FAQ readable on mobile.
- Do not require hover to reveal primary information.

## Implementation hygiene

- Reuse existing components when reasonable.
- Keep class naming and CSS ownership coherent with the detected stack.
- Do not create giant wrapper div chains when a small structural edit will do.
