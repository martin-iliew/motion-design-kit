# Execution Modes

Apply these rules before choosing patterns or editing code.

## fast

- Use compiled selectors only.
- Fix clear hierarchy and conversion misses first.
- Do not reorder the full page unless the current order is obviously broken.
- Prefer one strong change per weak section.

## balanced

- Use compiled selectors first, with selective fallback to `trends-overview.md`.
- Reorder sections when it clearly improves scan flow or trust.
- Add proof and objection handling when obviously missing.
- Keep output concise.

## premium

- Still stay conversion-first.
- Permit stronger composition changes, richer section framing, and more polished proof presentation.
- Do not add novelty-first design moves unless the user explicitly asks for them.
- Use slightly richer output if major restructuring happened.
