# Layout Patterns

Use these as stack-agnostic layout defaults.

## Hero

- 12-column desktop grid
- copy spans 5-6 columns
- media spans 5-6 columns
- proof row sits directly under CTA group

## Feature grids

- 2x2 or 3x2 modular grids beat novelty mosaics for most landing pages
- vary card span only when it reinforces hierarchy
- keep copy density even

## Proof rails

- use logos or metrics in a single calm row
- avoid oversized logo walls that dominate the page

## Pricing

- compare plans in parallel
- keep primary differentiators visible without expansion

## CTA bands

- pair the action with reassurance: demo, trial, migration support, security, or onboarding
